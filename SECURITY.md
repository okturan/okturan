# Security policy

## Supported version

Security fixes target the `main` branch and assets generated by its current
workflows. Historical output-branch assets and copied profile generators are not
patched in place.

## Reporting a vulnerability

Please use [GitHub's private vulnerability reporting form](https://github.com/okturan/okturan/security/advisories/new)
instead of opening a public issue. Include the affected workflow, generator, or
rendered asset; a reduced triggering input; and the security impact.

Useful reports include:

- script, markup, URL, or terminal-control injection through external profile,
  contribution, repository, or anime metadata;
- a workflow-permission, token, artifact, or output-branch weakness that could
  let untrusted content alter the profile;
- an unpinned or compromised dependency path with a demonstrated impact;
- a generated SVG or README link that crosses GitHub's expected rendering or
  navigation boundary.

The email address, professional links, public GitHub activity, and other details
deliberately shown in the README are public profile content, not disclosures.
Incorrect statistics, stale cards, broken badges, and visual defects can be
reported as normal bugs unless they enable a security boundary to be crossed.

Use synthetic metadata and redact tokens, credentials, and private account
details. The maintainer will coordinate validation, remediation, and disclosure
through the private advisory.
